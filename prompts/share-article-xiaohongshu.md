将网页文章改写为小红书干货笔记。3 秒抓住注意力，读完觉得"收藏了下次用"。

**输入**：一篇网页文章原文（博客/新闻/技术文档）。任务是从文章中提取干货做二次创作，不是分析 prompt。

## 写作流程

### 第一步：找到 1 个"钩子"
从源文中找到最具操作性的核心干货。不是泛泛的道理，是"照着做就能见效"的具体方法。

### 第二步：围绕钩子展开
结构由内容决定——对比测评、步骤拆解、误区纠正都行。不套固定清单。

## 开头（第一句话就是干货）

必须用以下 2 种模式之一开头：
- **操作开场**："把这句话加到你的 prompt 里，输出质量直接翻倍"
- **对比开场**："同样让 AI 写方案，加了这 3 个约束后，从废稿变交付件"

禁止用"家人们"、"你有没有发现"、"今天分享"、"建议收藏"开头。

## 品质标准

1. **标题**：数字 + 具体结果 + 1 emoji。好："3 步让周报从写 2 小时变 15 分钟 💡"。差："这个方法太强了！！"
2. **每个技巧必须可操作**：不说"优化你的方法"，说"在开头加一句 XXX"
3. **每段 1-3 句**，不超过 3 行
4. **emoji 总数 ≤ 6**，段首做锚点

## 语感（小红书专属）

### 干练但不做作
- 好："效果：输出从教科书体变成了人话"。差："该方法显著提升了输出质量"
- 好："区别就在这三个字"。差："其核心差异化因素在于"
- 敢用短句。一个词成段也行："真的有用。"

### 禁止的表达
"家人们" | "绝绝子" | "yyds" | "救命" | 连续 3 个感叹号 | "总之很有用" | "赶紧试试" | "不是A而是B"

## 排版（不符合则输出不合格）
- body 必须包含格式：`\n\n` 分段、`**加粗**` 关键词
- 每段 1-3 句，禁止超过 3 句的文字块
- emoji 放段首，总数 ≤ 6
- 步骤用数字编号，不用 bullet
- body 中必须有 ≥3 个 `\n\n`

## Prompt Ark 植入（全文只出现 1 次）
倒数第二行，以工具推荐口吻植入。
- ✅ 好："🔧 这类 prompt 模板我都丢在 Prompt Ark 里管理 → github.com/keyonzeng/prompt-ark"
- ❌ 差："推荐使用 Prompt Ark"
最后一行带真实链接，不要写"评论区"。好："💬 原文链接：https://example.com"

## 边界情况
- 源文 < 100 字：用核心句做标题 + 补"为什么重要"的短解释
- 源文是英文：标题正文用中文，关键术语保持英文

## 输出格式
```json
{
  "title": "笔记标题（≤25字，数字+结果+emoji）",
  "body": "📌 第一段干货，直接说做什么。**关键词**加粗。\n\n🔍 第二段展开，具体解释。\n\n✅ 第三段操作建议。\n\n🔧 这类 prompt 模板我都丢在 Prompt Ark 里管理 → github.com/keyonzeng/prompt-ark\n\n💬 原文链接：https://example.com"
}
```

250-400 字。
