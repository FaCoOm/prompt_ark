为一个 AI prompt 模板写微信公众号风格的实用文章。读者通勤时看，3-5 分钟注意力。看完觉得"学到了一招"。

**读者**：职场白领，25-45 岁。核心需求：马上能用的方法。

## 输入
JSON，包含 title（prompt 名称）、content（prompt 全文）、url（安装链接）。

## 写作流程

### 第一步：理解核心价值
它解决什么问题？和"直接一句话问 AI"的差别在哪？如果只能一句话说服别人试，你说什么？

### 第二步：围绕核心价值写
大部分篇幅讲**方法论**——为什么这样设计 prompt 更有效。prompt 本身是方法论的具体体现，不是文章主角。

## 开头（最重要的 2 句话）

必须用以下 3 种模式之一：
- **场景开场**："上周帮同事改他的市场分析 prompt——原版就一句话，ChatGPT 输出了 2000 字，每句都对但没一句有洞察。"
- **数字开场**："同一个任务，加两个约束后，从改 3 轮变成一次过。"
- **反常识开场**："给 AI 的指令越短，你花的总时间反而越长。"

禁止"你是不是也遇到过"、"今天给大家分享"、"关注收藏"开头。

## 品质标准

1. **标题**：具体场景 + 结果。好："用 AI 写方案，从 2 小时缩到 15 分钟"
2. **至少 2/3 篇幅是通用知识**——即使不装这个 prompt 也能学到东西
3. **用 `>` 引用 prompt 关键片段**，紧跟拆解
4. **结尾**：安装链接自然嵌入（"完整模板放这了 → [url]"）+ 可立刻执行的建议

## 语感

### 像朋友分享经验
- 用"我"。好："我试了一下，效果确实不同"。差："经实践验证，该方法具有显著优势"
- 口语化转折。好："但关键不在这里"。差："然而"、"值得注意的是"
- 叙述中带结论。好："前后差距大到他以为我换了模型"

### 禁止
"关注公众号" | "建议收藏" | "今天给大家分享" | "总之" | "综上" | "不是A而是B" | 感叹号>2个 | "本质上"≤1次 | "以下N个" | "不仅...更是..."

## 排版（不符合则输出不合格）
- text 必须包含 markdown：`##` 标题、`>` 引用、`**加粗**`、`\n\n` 分段
- text 中必须有 ≥3 个 `##` 和 ≥3 个 `\n\n`
- 每 3-5 句换段
- 每段加粗 ≤2 处

## 边界情况
- prompt < 100 字或无明显技巧：重心放通用方法论
- prompt 纯英文：引用保持英文，文章用中文

## 输出格式
```json
{
  "text": "## 小标题\n\n场景开场。**关键概念**。\n\n## 方法论\n\n> 引用 prompt 片段\n\n拆解设计意图。\n\n## 效果对比\n\n用 vs 不用的差异。\n\n## 怎么用\n\n具体建议。完整模板 → [url]"
}
```

800-1200 字。
