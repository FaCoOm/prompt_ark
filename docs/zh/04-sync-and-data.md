# 同步、数据与设置

## 选哪个同步方式？

| 方式 | 配置难度 | 容量 | 适合谁 |
|---|---|---|---|
| **仅本地** | 零配置（默认） | 当前设备存储 | 单设备私有使用 |
| **WebDAV** | 填服务器地址 + 密码 | 取决于服务器 | 注重隐私、私有部署（坚果云等） |
| **Obsidian Vault（WebDAV）** | 填 Vault 的 WebDAV 配置 | 取决于服务器 | Markdown 工作流、Obsidian 用户 |

## 仅本地（默认）
开箱即用，所有数据默认只保存在当前设备。

> ⚠️ 如果需要跨设备同步，请切换到 WebDAV 或 Obsidian Vault 同步。

## WebDAV 同步
1. **设置 → 同步** → 选 **WebDAV**
2. 填 **WebDAV 地址**、**账号**、**应用密码**
3. 点 **立刻同步 WebDAV**

> 主流 WebDAV 服务：坚果云、Nextcloud、群晖

## Obsidian Vault 同步
1. **设置 → 同步** → 选 **Obsidian Vault（WebDAV）**
2. 填 Vault 的 **WebDAV 地址**、**账号**、**应用密码** 和目标子文件夹
3. 点 **立刻同步 Obsidian**

## JSON 导出 / 导入
- **导出**：设置齿轮 → 导出 → 下载 `.json` 文件
- **导入**：粘贴（原始文本/JSON）或 URL（从网址抓取）

## GitHub Token 有什么用？

设置中的 GitHub Token 主要用于 GitHub 链接导入相关能力：

| 能力 | 没有 Token | 有 Token |
|---|---|---|
| GitHub URL 导入 | 60 次/小时（容易触发限制） | **5000 次/小时** |
| GitHub 仓库扫描 | 更容易撞限流 | 扫描大型仓库更稳定 |

## 自动保存
所有设置页（全部 Tab）都是 600ms 防抖自动保存，无需任何「保存」按钮。仅添加/编辑模型供应商时需要点「保存」按钮确认。
