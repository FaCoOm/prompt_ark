# 别再把 AI 提示词存备忘录了！这款开源神器让你的 Prompt 随叫随到

> 完全免费，开源，4000 行代码，适配 14 个 AI 平台。我做的，今天正式开源。

---

## 👋 先说说我为什么做这个

我每天至少用 5 个 AI 平台。ChatGPT 写文档，Claude 审代码，DeepSeek 做中文问答，Kimi 读论文，Gemini 做搜索。

用得越多，越觉得有个事情不对劲——

### 场景一：找不到了

上周写了一个特别好用的提示词，让 ChatGPT 帮你把会议记录整理成"决策 + 待办 + 遗留问题"三段式。当时效果惊人。

今天又要整理会议纪要了。

你打开备忘录，搜"AI"——出来 47 条结果。搜"会议"——出来 12 条。你翻了三分钟，不确定是哪一条了，其中有几条是半成品，有几条是别人发给你的，还有一条是你凌晨三点写的、只有半句话。

算了，重新写一个吧。

### 场景二：忘了改占位符

你有一个万能的冷启动邮件模板，里面写着"尊敬的 `[客户名称]`，我是 `[你的公司]` 的 `[你的职位]`……"

复制粘贴，发了出去。

十分钟后你才意识到——你给客户发了一封以"尊敬的 [客户名称]"开头的邮件。

### 场景三：分享不了

你调教了一组提示词，覆盖日报、周报、OKR 拆解、绩效自评。效果很好，同事都想要。

然后你打开微信，发了第一条——格式乱了。发了第二条——对方说"你上面那条是不是没发完？"。发到第四条你放弃了，说"算了我发你个文档"。

于是你打开 Notion，新建了一个页面，复制粘贴了四遍，调了半天格式，发了个链接。

对方说："我没有 Notion 账号。"

### 场景四：在错误的地方管理

你终于受不了，在手机上建了一个"AI提示词"的备忘录文件夹。

然后你发现——每次要用提示词的时候，你要：**打开手机 → 找到备忘录 → 找到那条 → 复制 → 切到电脑 → 粘贴到 ChatGPT**。

六步。就为了用一条你自己写的东西。

**这个流程不是"管理提示词"，是"折磨自己"。**

---

我意识到，问题的本质不是"没有地方存提示词"——你有备忘录、有 Notion、有微信收藏。

问题是：**没有一个工具能在你用 AI 的那个地方，直接让你调出提示词。**

所有其他工具的逻辑都是：先去别的地方找 → 复制 → 切回来 → 粘贴。

我想要的很简单：**我在 ChatGPT 聊天，聊天框旁边就有个按钮，点一下就把提示词填进去。**

于是我做了 **Prompt Ark**（提示词方舟）。

---

## 它到底做了什么

打开任何 AI 平台，你会看到聊天框旁边多了两个按钮：

- **✨** — 打开提示词库，选一条直接注入
- **⚡** — 一键润色 / 翻译 / 总结 / 扩写 / 解释

就这么简单。但让这两个按钮在 14 个平台上都正常工作，我写了 1000 多行代码。

## 说说里面我觉得最有意思的技术细节

### 每个平台都需要不同的注入方式

往聊天框里发一段文字，听起来很简单对吧？

实际上——

- **ChatGPT 和 Claude** 用 ProseMirror 富文本编辑器，你直接改 DOM 它不认，需要用 `execCommand('insertText')`
- **Gemini 和通义千问** 用 React 管理的 textarea，你直接设 `.value` 它不认，需要绕过 React 的内部 value tracker，找到原型链上的 setter 才行
- **NotebookLM** 的输入框藏在 Shadow DOM 里，普通的 `querySelector` 根本找不到，需要递归穿透每一层 shadow boundary

而且注入之后还要模拟一整套事件链（input → change → keydown → keyup），不然发送按钮不会亮起来。

我给每个平台都调了按钮的位置偏移。比如 NotebookLM 右边有严格的 overflow 裁剪，按钮放太靠右会被吃掉，我专门改成了 `right: 90px`。

**这些东西用户看不到，但"就是好用"的感觉来自这些地方。**

### 搜索框和聊天框要区别对待

代码会检测输入框的 placeholder——如果包含 "search"、"research"、"搜索"，就只显示 ✨（提示词选择器），不显示 ⚡（快捷指令）。因为你不需要在搜索框里"润色"。

### 页面上下文可以当变量用

你可以在提示词里写 `{{page_title}}`、`{{selected_text}}`、`{{page_url}}`。

比如写一条提示词：

```
请根据这篇文章的内容，写一个 200 字的摘要：

标题：{{page_title}}
正文：{{selected_text}}
```

在任何网页上选中一段文字，切到 ChatGPT，点 ✨ 选这条提示词——变量自动填充。支持**跨标签页**，缓存 10 分钟。

## 100 条内置提示词不是凑数的

我花了很多时间打磨这 100 条预装提示词（50 英文 + 50 中文）。每一条都遵循三个原则：

**1. 有结构化输出格式**

比如「会议纪要整理器」，不是简单说"帮我总结会议"，而是指定了精确的输出框架：

```
**🎯 Decisions Made**
- [numbered list]

**📋 Action Items**
| Owner | Task | Deadline |
|---|---|---|

**❓ Open Questions**
- [numbered list]
```

**2. 有负向约束**

每条提示词都有"不要做什么"。比如：
- 会议纪要："Do NOT add information not present in the notes"（不要添加笔记里没有的信息）
- 理财规划："Do NOT give generic advice like 'save more money'"（不要说"多存钱"这种废话）

这是防止 AI 幻觉和敷衍的关键。

**3. 都有 `{{变量}}`**

每条都带动态变量，使用时弹出表单填写。同一条提示词，每次填不同的值，不用手动改占位符。

我最得意的几条：
- 📊 **SWOT 分析师** — 填个公司名，直接出四象限分析
- 🐛 **调试助手** — 粘贴报错信息，按 5 步法排查
- ✍️ **小红书风格改写** — 把任何文字改成爆款文风
- 🧠 **苏格拉底教学法** — 不直接回答，用提问引导你自己找到答案

## AI 优化器是我最喜欢的功能

你写了一条提示词，觉得还行但不够专业？点「✨ 优化」，AI 帮你改写成三个版本：

| 版本 | 思路 |
|---|---|
| **精简版** | 砍掉一切冗余，只留最核心的指令 |
| **增强版** | 补上输出格式、边界条件和约束 |
| **专业版** | 加入角色设定、结构化分段、思维链 |

我写了一个专门的 system prompt 来驱动这个功能，核心原则是：

- **首因效应** — 把核心指令放在最前面，因为模型对前面的 token 权重更高
- **负向约束** — 用"不要做什么"替代"请做什么"，效果更好
- **噪音清理** — 砍掉对模型行为没有影响的废话（恭维、威胁、元评论）

每个版本都有**逐行 Diff 对比**，改了哪里一目了然。一键采纳或放弃。

最重要的：**这个功能不需要 API Key。** 它默认用你浏览器的 Gemini 登录会话。装完就能用。

## 一些你可能好奇的问题

**Q: 需要配置 API Key 吗？**
不需要。所有 AI 功能默认用 Gemini Web（你浏览器登录了 gemini.google.com 就行）。想用自己的 GPT-4o 或 DeepSeek API 也可以，在设置里加一个就行。

**Q: 数据存在哪？安全吗？**
全部存在你自己的浏览器里。不经过任何第三方服务器。同步方式三选一：Chrome 原生同步、GitHub Gist、WebDAV（坚果云等）。

**Q: 可以分享给同事吗？**
可以。一键发布到 GitHub Gist，自动生成预览页面。还可以多条提示词打包成「Prompt Pack」分享。

**Q: 支持哪些浏览器？**
Chrome 和 Edge 都完整支持。
